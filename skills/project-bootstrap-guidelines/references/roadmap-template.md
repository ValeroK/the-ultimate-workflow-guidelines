# ROADMAP

> Canonical skeleton for the project-level execution roadmap produced in Phase 3 of `project-bootstrap-guidelines`. Distinct from per-feature `PLAN-<feature>.md` files (which are produced by `the-ultimate-workflow-guidelines`).
> This document holds the **milestone view**. Update it whenever a milestone closes or scope shifts.

## How to read this

- **Milestones** are ordered phases of work, each with a user-visible outcome.
- **Tasks** are the discrete pieces of work that close a milestone.
- **Dependencies** note tasks / milestones that must land first.
- Check items off as they complete; the milestone closes when all tasks are done.
